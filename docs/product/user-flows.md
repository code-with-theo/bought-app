# 买过｜MVP 用户流程

## 0. 体验基线

- 产品是私人的购物记忆库，不是电商或公开社区。
- 首次基础记录只要求：商品名称、商品照片、分类；目标在十几秒内完成。
- 评价可以稍后补；“会复购 / 不会复购 / 再看看”始终是最醒目的结论。
- 移动端优先，底部导航和主操作不能被安全区域遮挡。

## 1. 首次打开与演示数据

1. 用户首次打开，应用检查本地是否已有初始化标记。
2. 无标记时写入 6–8 条演示记录并显示首页；不弹出打断式引导。
3. 首页呈现累计记录、值得复购、最近记录和等待评价，使用户立即理解价值。
4. 用户可在“我的”中清除演示数据；仅删除标记为演示的记录，保留用户新增记录。

## 2. 添加商品

1. 用户从底部中央添加按钮或首页“记录商品”进入添加页。
2. 输入商品名称，选择/拍摄一张照片，选择分类。
3. 图片选中后立即压缩并显示预览；可替换或删除后重选。
4. 用户可展开“补充一下”填写品牌、价格、购买日期、渠道、使用状态和备注。
5. 点击“保存这次购买”；缺少必填项时在对应字段下显示原因并定位到首个错误。
6. 保存成功显示 Toast，进入该商品详情页；初始评价为空、复购结论为“再看看”。

## 3. 上传、预览与替换图片

1. 点击图片区域唤起系统文件/相册选择器，限定图片类型。
2. 读取失败或压缩失败时保留原表单内容，显示可恢复的错误提示和“重新选择”。
3. 预览状态提供“换一张”和“删除”；删除后回到待上传状态，不能保存。
4. 展示图片加载失败时显示带分类色块和商品名称首字的备用图，不阻塞查看或编辑。

## 4. 保存与商品详情

1. 详情页展示照片、名称、分类、购买信息和备注；缺失的选填信息不显示占位横线。
2. 未评价记录在顶部显示轻提示“用过以后，回来留一句感受吧”。
3. 评分、复购结论和一句话评价可直接在详情页完成。
4. 保存评价后立即更新详情、首页和物品库数据，显示“已记下你的感受”。

## 5. 延迟评价

1. 用户可从首页“等待评价”卡片或详情页进入评价。
2. 评价顺序不限：可只选结论、只评分或只写评价；任一操作即时保存并 Toast 提示。
3. “等待评价”定义为：评分未填写，或复购结论仍为“再看看”；一句话评价可为空，不影响该判断。
4. 有明确“会复购”结论的记录进入首页“值得复购”与底部“复购”预设视图。

## 6. 评分与复购结论

1. 用户点击 1–5 星设置整数评分；再次点击当前星不取消评分。
2. 评分旁同步显示：1 很差、2 不太行、3 一般、4 不错、5 真香。
3. 复购结论三选一：会复购、不会复购、再看看；选中态持久且显著。
4. “不会复购”不等同删除；它计入“避雷数量”，并可在物品库筛选到。

## 7. 搜索、筛选和排序

1. 用户进入“物品库”，默认按最近记录倒序显示全部记录；筛选、排序与视图状态仅需在当前会话保留，无需刷新后恢复。
2. 快捷筛选为：全部、会复购、不会复购、等待评价；任一时刻只能有一个快捷筛选。
3. 搜索商品名称（同时匹配品牌，如已填写）时结果实时更新，输入防抖不超过 300ms。
4. 筛选面板支持分类、复购结论、评分；支持按最近记录或评分排序，以及网格/列表视图切换。
5. 结果区同时体现是否买过、最近购买时间、评分、复购结论和一句评价（无评价时显示“还没写感受”）。
6. 有条件无结果时提供“清除筛选”，无任何记录时引导“记录第一件商品”。

## 8. 编辑与删除

1. 详情页“编辑”带入现有数据，编辑保存后返回详情并显示“已更新”。
2. 编辑中放弃修改时返回详情；已填写内容不自动覆盖原记录。
3. 删除入口在详情页二级操作中。点击后出现自定义确认弹窗，明确商品名和“删除后无法恢复”。
4. 取消关闭弹窗且不改变数据；确认删除后 Toast 提示，返回来源页（无来源则物品库）。

## 9. 导出与清除数据

1. “我的”可导出全部记录为 JSON；导出文件包含文本字段与图片数据/可恢复的图片引用，具体格式由数据层确认。
2. “清除演示数据”只在存在演示记录时展示，并二次确认。
3. “清空全部数据”始终二次确认，文案说明会同时清除照片和评价；确认后展示全空首页。
4. 两类清除均不得使用浏览器原生 alert/confirm。
