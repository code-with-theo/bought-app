# 买过｜MVP 验收条件

## 核心记录

- [ ] 首次打开可看到 6–8 条中文演示数据，覆盖食品、服饰、美妆、家居、数码且包含不同评分与复购结论。
- [ ] 用户只填写名称、照片、分类即可完成保存；保存后进入新商品详情。
- [ ] 可从系统相册选择图片、看到预览、替换或删除图片；大图不会直接以原尺寸无限制写入本地存储。
- [ ] 图片格式、大小、读取或压缩失败均给出可恢复错误；不会写入缺少图片的损坏记录。
- [ ] 保存期间主提交按钮不可重复触发，不产生重复记录。
- [ ] 所有选填字段可留空；刷新后新增和编辑记录仍保留。

## 评价与详情

- [ ] 详情页可设置 1–5 星，并显示匹配的中文评分文案。
- [ ] 三种复购结论互斥且清晰：会复购、不会复购、再看看。
- [ ] 用户可延后评价；未评价记录在首页“等待评价”出现，明确会复购记录在复购页出现。
- [ ] “等待评价”严格定义为评分为空或复购结论为“再看看”；一句话评价为空不单独构成等待评价。
- [ ] 一句话评价与快捷标签可编辑并持久化。
- [ ] 删除操作有自定义二次确认；取消不会修改数据，确认后记录在所有页面消失。

## 发现与管理

- [ ] 物品库支持名称/品牌搜索、分类/结论/评分筛选、最近/评分排序、网格/列表切换和清除筛选。
- [ ] 搜索结果展示是否买过、最近购买时间、评分、复购结论和一句评价（或未评价提示）。
- [ ] 复购页只显示“会复购”商品。
- [ ] 我的页统计总记录、会复购、避雷数量准确；可导出本地记录。
- [ ] 清除演示数据不删除用户数据；清空全部数据会删除所有本地记录，两者均二次确认。

## 体验与鲁棒性

- [ ] 所有页面均有匹配的空、加载和错误状态；错误不导致白屏或崩溃。
- [ ] 图片加载失败使用备用展示；异常/缺字段旧数据仍可正常读取或安全忽略。
- [ ] 无浏览器原生 alert/confirm；保存、删除、导出等操作提供 Toast 或等效反馈。
- [ ] 375px 宽度下底部导航、表单和主按钮可用且不遮挡；桌面端内容不会无限拉宽。
- [ ] 所有可交互按钮提供可见的按下、悬停和禁用状态；键盘可操作表单。
- [ ] 无效详情 URL、缺失字段或损坏本地存储均安全降级，不出现白屏；图标按钮具备可访问名称，状态不只依赖颜色。

## 验收流程

1. 初次打开，确认首页有完整演示数据与明确的“记录商品”入口。
2. 上传照片、填写名称与分类，保存并进入详情。
3. 设置 4 星、选择“会复购”、写一条评价；回首页确认内容同步更新。
4. 在物品库按该分类找到商品，再筛选“会复购”确认出现。
5. 编辑评价，刷新页面，确认修改仍在。
6. 删除该记录，确认弹窗存在；确认后在详情、首页和物品库均不可见。
7. 以缺失字段的旧记录和失效图片进行检查，确认页面不崩溃。
