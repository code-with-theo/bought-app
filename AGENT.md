# 一、产品定位

「买过」是一个私人购物记忆库。

用户可以记录自己买过的吃的、穿的、用的，为商品拍照、分类、打分，并给出明确的复购结论：

- 会复购
- 不会复购
- 再看看

产品的核心价值是：

下次看到同一个或类似商品时，用户能够快速知道自己是否买过、使用感受如何、还要不要再买。

它不是电商平台，也不是公共测评社区，而是一个简单、好看、方便使用的私人购物记录工具。

# 二、开发目标

先完成一个移动端优先、可以实际操作的 MVP。

技术栈：

- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui
- Lucide Icons
- 使用 IndexedDB 或合适的本地存储方案保存数据
- 暂时不要求登录和云端数据库
- 保持数据访问层独立，方便后续迁移至 Supabase
- 支持安装为 PWA；如果本阶段工作量过大，可以先预留配置

请使用稳定版本，不要使用不必要的实验性依赖。

# 三、视觉设计

整体风格：

- 简单、温柔、有生活感
- 像私人收藏夹，而不是库存管理软件
- 高颜值但不过度装饰
- 移动端优先
- 大面积留白
- 圆角卡片
- 柔和阴影
- 商品照片是主要视觉元素
- 页面不能像后台管理系统

建议色彩：

- 背景：米白色或暖白色
- 主色：珊瑚橙红
- 辅助色：低饱和鼠尾草绿
- 正文：接近黑色的暖灰
- 卡片：白色

避免：

- 大面积渐变
- 玻璃拟态
- 过多图表
- 复杂导航
- 密集表格
- 过度卡通化
- 页面中堆砌大量功能

所有界面使用中文。

# 四、第一版页面

## 1. 首页

首页需要包括：

- 顶部欢迎语和 App 名称「买过」
- 搜索入口
- 本月或累计记录数量
- “值得复购”横向商品卡片
- “最近记录”商品列表
- “等待评价”区域
- 固定在底部或右下角的醒目“记录商品”按钮
- 空数据状态

首页文案示例：

“记下每一次选择，让下一次购买更值得。”

商品卡片显示：

- 商品照片
- 商品名称
- 分类
- 评分
- 复购结论
- 一句话评价
- 购买日期

## 2. 添加记录页

完成一次基础记录应尽量控制在十几秒内。

必填字段：

- 商品名称
- 商品照片
- 分类

选填字段：

- 品牌
- 价格
- 购买日期
- 购买渠道
- 当前状态
- 一句话备注

预设分类：

- 吃的
- 穿的
- 护肤美妆
- 家居日用
- 数码电器
- 其他

当前状态：

- 还没开始用
- 使用中
- 已经用完

需要支持：

- 手机相册选择图片
- 图片预览
- 修改和删除图片
- 表单验证
- 保存成功反馈
- 保存后跳转至商品详情页

第一版先把图片存储在浏览器本地。请注意图片压缩和浏览器存储空间，避免直接保存尺寸过大的原图。

## 3. 商品详情与评价页

页面显示完整商品资料。

评价区域包括：

### 综合评分

1 至 5 星，支持半星不是必需的。

评分旁边显示对应文案：

- 1 星：很差
- 2 星：不太行
- 3 星：一般
- 4 星：不错
- 5 星：真香

### 复购结论

三个明显按钮：

- 会复购
- 不会复购
- 再看看

这是页面中最重要的操作，应当比其他字段更醒目。

### 一句话评价

提供文本输入，同时支持快捷标签：

- 性价比高
- 颜值高
- 好用
- 不耐用
- 不舒服
- 太贵
- 容易坏
- 会回购
- 踩雷

支持编辑和删除商品记录。

删除操作必须二次确认。

## 4. 我的物品库

使用图片卡片或紧凑列表展示，不使用传统表格。

支持：

- 商品名称搜索
- 分类筛选
- 复购结论筛选
- 评分筛选
- 按最近记录排序
- 按评分排序
- 在网格和列表视图之间切换
- 清除筛选条件
- 无搜索结果状态

顶部提供快捷筛选：

- 全部
- 会复购
- 不会复购
- 等待评价
